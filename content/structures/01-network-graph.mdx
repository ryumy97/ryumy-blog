# 네트워크 그래프 - 관계와 연결의 시각화

네트워크 그래프는 복잡한 관계와 연결을 시각화하는 가장 효과적인 방법 중 하나입니다. 소셜 네트워크, 생물학적 시스템, 인터넷 구조 등 다양한 분야에서 활용되며, 데이터 간의 관계를 직관적으로 이해할 수 있게 해줍니다.

## 네트워크 그래프의 기본 구조

네트워크 그래프는 **노드(Node)**와 **엣지(Edge)**로 구성됩니다:

- **노드**: 개체나 엔티티를 나타내는 점
- **엣지**: 노드 간의 관계나 연결을 나타내는 선

### 네트워크 그래프의 장점

- **관계 시각화**: 복잡한 관계를 한눈에 파악
- **패턴 발견**: 숨겨진 구조와 커뮤니티 식별
- **중앙성 분석**: 중요 노드와 연결성 파악
- **동적 표현**: 시간에 따른 변화 추적 가능

## 실제 구현 예제

다음은 React와 D3.js를 사용한 네트워크 그래프 예제입니다:

```jsx
import React, { useEffect, useRef } from "react";
import * as d3 from "d3";

const NetworkGraph = ({ data }) => {
  const svgRef = useRef();

  useEffect(() => {
    if (!data || !svgRef.current) return;

    const svg = d3.select(svgRef.current);
    const width = 800;
    const height = 600;

    // Clear previous content
    svg.selectAll("*").remove();

    // Create force simulation
    const simulation = d3
      .forceSimulation(data.nodes)
      .force(
        "link",
        d3
          .forceLink(data.links)
          .id((d) => d.id)
          .distance(100)
      )
      .force("charge", d3.forceManyBody().strength(-300))
      .force("center", d3.forceCenter(width / 2, height / 2))
      .force("collision", d3.forceCollide().radius(30));

    // Create links
    const links = svg
      .append("g")
      .selectAll("line")
      .data(data.links)
      .enter()
      .append("line")
      .attr("stroke", "#999")
      .attr("stroke-opacity", 0.6)
      .attr("stroke-width", (d) => Math.sqrt(d.value));

    // Create nodes
    const nodes = svg
      .append("g")
      .selectAll("circle")
      .data(data.nodes)
      .enter()
      .append("circle")
      .attr("r", (d) => d.size || 8)
      .attr("fill", (d) => d.color || "#69b3a2")
      .attr("stroke", "#fff")
      .attr("stroke-width", 2)
      .call(
        d3
          .drag()
          .on("start", dragstarted)
          .on("drag", dragged)
          .on("end", dragended)
      );

    // Add labels
    const labels = svg
      .append("g")
      .selectAll("text")
      .data(data.nodes)
      .enter()
      .append("text")
      .text((d) => d.name)
      .attr("font-size", "12px")
      .attr("dx", 12)
      .attr("dy", 4);

    // Update positions on simulation tick
    simulation.on("tick", () => {
      links
        .attr("x1", (d) => d.source.x)
        .attr("y1", (d) => d.source.y)
        .attr("x2", (d) => d.target.x)
        .attr("y2", (d) => d.target.y);

      nodes.attr("cx", (d) => d.x).attr("cy", (d) => d.y);

      labels.attr("x", (d) => d.x).attr("y", (d) => d.y);
    });

    // Drag functions
    function dragstarted(event, d) {
      if (!event.active) simulation.alphaTarget(0.3).restart();
      d.fx = d.x;
      d.fy = d.y;
    }

    function dragged(event, d) {
      d.fx = event.x;
      d.fy = event.y;
    }

    function dragended(event, d) {
      if (!event.active) simulation.alphaTarget(0);
      d.fx = null;
      d.fy = null;
    }

    // Add hover effects
    nodes
      .on("mouseover", function (event, d) {
        d3.select(this)
          .attr("r", (d.size || 8) * 1.5)
          .attr("fill", "#ff6b6b");
      })
      .on("mouseout", function (event, d) {
        d3.select(this)
          .attr("r", d.size || 8)
          .attr("fill", d.color || "#69b3a2");
      });
  }, [data]);

  return (
    <svg
      ref={svgRef}
      width="800"
      height="600"
      style={{ border: "1px solid #ccc" }}
    />
  );
};

export default NetworkGraph;
```

## 네트워크 그래프의 다양한 유형

### 1. 무방향 그래프 (Undirected Graph)

노드 간의 관계가 양방향인 경우

### 2. 방향 그래프 (Directed Graph)

노드 간의 관계가 한 방향인 경우

### 3. 가중 그래프 (Weighted Graph)

엣지에 가중치가 있는 경우

### 4. 다중 그래프 (Multigraph)

두 노드 간에 여러 엣지가 있는 경우

## 레이아웃 알고리즘

### 1. Force-Directed Layout

가장 일반적인 레이아웃으로, 물리적 힘을 시뮬레이션합니다.

### 2. Circular Layout

노드들을 원형으로 배치하여 대칭성을 강조합니다.

### 3. Hierarchical Layout

계층 구조를 반영하여 트리 형태로 배치합니다.

### 4. Radial Layout

중심 노드에서 방사형으로 확장하는 레이아웃입니다.

## 디자인 고려사항

### 노드 디자인

- **크기**: 중요도나 연결 수에 비례
- **색상**: 카테고리나 그룹을 구분
- **모양**: 노드의 유형을 구분
- **라벨**: 가독성을 위한 텍스트 배치

### 엣지 디자인

- **두께**: 관계의 강도나 가중치
- **색상**: 관계의 유형을 구분
- **스타일**: 실선, 점선, 화살표 등
- **투명도**: 시각적 혼잡도 조절

### 상호작용

- **드래그**: 노드 위치 조정
- **줌**: 확대/축소 기능
- **필터링**: 특정 노드나 엣지 숨김
- **하이라이트**: 관련 요소 강조

## 성능 최적화

### 렌더링 최적화

- **WebGL**: 대용량 네트워크 처리
- **Canvas**: 빠른 렌더링
- **SVG**: 벡터 기반 고품질

### 데이터 최적화

- **데이터 구조**: 효율적인 노드/엣지 표현
- **메모리 관리**: 불필요한 객체 제거
- **알고리즘**: 효율적인 레이아웃 계산
